GOV.UK Prototyping Kit

What is the Prototyping Kit

{% from "govuk/components/inset-text/macro.njk" import govukInsetText %} {{ govukInsetText({ text: "This section is based on the original prototype kit tutorial and hence contains public sector information licensed under the Open Government Licence v3.0." }) }}

When making high fidelity prototypes, we want them to look like the service we're designing. For central government services, we want to make them look like they belong to GOV.UK.

To do this, we can use the GOV.UK Prototype Kit. It lets us make hi-fi prototypes to use for testing, and sharing with developers to help them understand what needs to be achieved by the final service.

It contains templates for pages and components, as well as guides on how to use them to develop a working prototype.

How to use the Prototyping Kit

The Prototyping Kit is intended to help make the process of high-fidelity prototyping easier, as you develop your low-fi prototypes for further testing.

The prototyping process is about feedback and iteration, not making the perfect product first time. Think of your prototypes as a tool for conversation with users and stakeholders, rather than completed products. It's generally better to share early and often.

It can be helpful to develop several different versions, in order to prototype in parallel.

Tech in The PSC

The Kit has some prerequisites which have to be in place to use it, including Node and python.

Code written using the Kit should also be saved in the open, using Git (64-bit). This lets you work collaboratively with your team.

In The PSC, we use the Prototyping Kit through Visual Studio Code, a lightweight development environment designed for writing and running code. It presents code in an easy to read form to help you write code quickly and avoid errors.

See the help and guidance links below if you're unsure about how to install or use any of these tools. You can also discuss them with your project team.
